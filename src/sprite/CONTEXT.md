# Sprite — context

## Build pipeline

`pnpm svg:build-sprite` reads SVG sources from `src/sprite/icons/` and emits
the generated sprite (`sprite.<hash>.svg`, `registry.json`, `README.md`) to
`public/icons/`. Those outputs are gitignored. The icon-name type declaration
`public/icons/name.d.ts` is also generated by the script but **is** tracked
— commit it alongside any icon add/remove/rename so type-checking stays
consistent for everyone.

`build:next` invokes `svg:build-sprite` automatically; for local iteration
you can run it directly.

Some icons under `icons/` are also imported directly as React components
rather than rendered via `SpriteIcon`. The main reason is the toolkit —
components in `src/toolkit/` cannot rely on the sprite runtime because
downstream consumers of `@blockscout/ui-toolkit` will not have the sprite
asset (see `src/toolkit/CONTEXT.md`). Any icon used by a toolkit component
must therefore be importable as a React component, not only via the sprite.
